Define an ordinal set to have an advanced control ordering the jobs. The entries in the ordinal set have to be separated by the "<" character.
Each match of the ordinal set in a job's full name, would insert the ordinal number in front of the matching entry, when comparing job names.

Example

Precondition:

  • Jobs:
    • HelloApp/Deploy_to_Test
    • DemoApp/Deploy_to_Test
    • DemoApp/Deploy_to_Prod
  • Ordinal Set:
    • CI_Pipeline < Deploy_to_Test < Deploy_to_Prod

      • I.e. string "CI_Pipeline" has ordinal number "#0", "Deploy_to_Test" has "#1", and "Deploy_to_Prod" has "#2"
Therefore the following is compared:
  • HelloApp/#1_Deploy_to_Test
  • DemoApp/#1_Deploy_to_Test
  • DemoApp/#2_Deploy_to_Prod
Therefore this results in job order:
  1. DemoApp/Deploy_to_Test
  2. DemoApp/Deploy_to_Prod
  3. HelloApp/Deploy_to_Test
